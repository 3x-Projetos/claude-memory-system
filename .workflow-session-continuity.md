# Workflow: Continuidade de Sessões

Este workflow documenta como manter contexto e continuidade entre diferentes sessões com Claude.

## Propósito

Permitir que cada nova sessão tenha acesso ao contexto da sessão anterior, possibilitando:
- Retomar trabalho de onde parou
- Ver pendências e próximos passos
- Manter consistência em projetos de longo prazo
- Evitar repetição de contexto

---

## Como Funciona

### 1. Ao Iniciar Nova Sessão

**Comando**: `/start`

**Processo automático**:
1. Carrega `.claude-memory.md` (ferramentas disponíveis)
2. Busca arquivo de log mais recente (YYYY.MM.DD.md)
3. Lê seções "Próximos Passos" e "Notas"
4. Apresenta resumo da última sessão
5. Lista tarefas pendentes ([ ] não marcadas)
6. Pergunta se deve continuar de onde parou ou iniciar nova atividade

### 2. Continuando de Onde Parou

Se o usuário escolher **continuar**:
- Claude usa as pendências como ponto de partida
- Retoma o contexto do que estava sendo trabalhado
- Mantém foco nas tarefas listadas em "Próximos Passos"

Se o usuário escolher **nova atividade**:
- Claude está ciente do histórico mas inicia novo foco
- Pode referenciar trabalho anterior se relevante

## Estrutura de Logs para Continuidade

### Seções Críticas

**Próximos Passos**:
```markdown
### Próximos Passos

- [ ] Tarefa pendente 1
- [ ] Tarefa pendente 2
- [x] Tarefa já concluída
```

**Notas**:
```markdown
### Notas

Contexto importante, decisões tomadas, insights que serão úteis na próxima sessão.
```

### Boas Práticas

1. **Próximos Passos**: Seja específico e acionável
   - ❌ "Melhorar código"
   - ✅ "Refatorar função processData() em src/utils.js"

2. **Notas**: Registre decisões e razões
   - Por que escolheu determinada abordagem
   - Bloqueios encontrados
   - Dependências descobertas

3. **Contexto suficiente**: A próxima sessão deve entender sem precisar ler tudo
   - Inclua referências de arquivos
   - Mencione próximos passos óbvios

## Exemplos de Uso

### Exemplo 1: Retomando Projeto

**Início da sessão**:
```
Usuário: /start

Claude:
Ferramentas carregadas: 4 disponíveis
- Organização de notas
- Session logging
- Session continuity

Última sessão: 2025-11-15
Tópico: Setup de sistema de memória e workflows

Pendências:
- [ ] Reiniciar Claude CLI para ativar slash commands
- [ ] Testar /start em nova sessão
- [ ] Validar workflow de organização

Deseja continuar de onde paramos ou iniciar nova atividade?

Usuário: Continuar

Claude: Ótimo! Vejo que você ainda precisa testar os slash commands.
Já reiniciou o CLI? Posso ajudar com os testes.
```

### Exemplo 2: Nova Atividade com Contexto

```
Usuário: /start

Claude: [mesmo resumo acima]

Usuário: Nova atividade - quero criar um script de automação

Claude: Entendido! Vamos trabalhar no script de automação.
Observo que você tem um sistema de workflows estabelecido.
Quer que eu crie um workflow documentado para este script também?
```

## Comandos Relacionados

### `/start`
Inicia sessão com contexto da sessão anterior

### `/end`
Finaliza sessão criando log estruturado para próxima sessão

### `/memory`
Consulta ferramentas disponíveis (útil durante a sessão)

### `/resume` (futuro)
Comando dedicado apenas para retomar última atividade sem escolha

---

## Benefícios

1. **Continuidade Natural**: Não precisa reexplicar contexto
2. **Produtividade**: Retoma direto de onde parou
3. **Rastreabilidade**: Histórico completo do trabalho
4. **Flexibilidade**: Pode continuar ou mudar de foco
5. **Memória Distribuída**: Logs servem como memória persistente
