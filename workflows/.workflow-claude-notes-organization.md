# Workflow: Organização de Notas com Claude

Este documento descreve o workflow estabelecido para organização de notas diárias com assistência do Claude.

## Estrutura de Arquivo de Notas

### Blocos Principais
Os arquivos de notas devem conter os seguintes blocos, separados por `---`:

1. **[raw]** - Notas brutas/originais do dia
   - Primeira linha do arquivo
   - Contém pensamentos, ideias e anotações sem filtro
   - Formato livre, bullets, tópicos

2. **[prompt]** - Instrução para processamento
   - Define a ação que Claude deve executar
   - Pode ser customizada para diferentes tipos de análise
   - Exemplo padrão: identificar tópicos, organizar e sugerir prioridades

3. **[organized]** - Resultado do processamento
   - Análise estruturada das notas
   - Identificação e organização de tópicos
   - Prioridades sugeridas com próximos passos

## Workflow de Execução

### Comando Padrão
```
leia o [caminho-do-arquivo].md e execute a ação contida no penúltimo bloco de texto, endereçado como [prompt]. Registre o retorno no local demarcado [here/organized], mantendo a tag.
```

### Processo
1. Claude lê o arquivo especificado
2. Identifica o bloco `[prompt]`
3. Executa a ação descrita no prompt sobre o conteúdo `[raw]`
4. Registra o resultado no bloco `[organized]` (ou outra tag indicada)
5. Mantém as tags e separadores `---` para organização visual

## Convenções

### Tags
- **Sempre em minúsculas**: `[raw]`, `[prompt]`, `[organized]`
- Tags customizadas podem ser criadas conforme necessidade
- Sugestões: `[analysis]`, `[response]`, `[processed]`

### Separadores
- Usar `---` entre blocos para separação visual clara
- Padrão: `[raw]` --- `[prompt]` --- `[organized]`

### Estrutura de Saída
Quando organizar notas, incluir:
- **Topics Identified & Organized** - Lista de tópicos encontrados com descrições
- **Suggested Priorities** - Prioridades ordenadas com bullets de próximos passos

## Exemplo de Arquivo

```markdown
[raw]

Minhas notas do dia aqui...
- Tópico 1
- Tópico 2
- Ideias soltas

---
[prompt]
Identifique os tópicos e organize-os.
Sugira prioridades com próximos passos.

---
[organized]

## Topics Identified & Organized
...

## Suggested Priorities
...
```

## Notas de Implementação
- Este workflow pode ser adaptado para diferentes tipos de análise
- O bloco `[prompt]` pode conter instruções customizadas
- Claude pode sugerir tags apropriadas ao finalizar o processamento
- Manter consistência nas tags e estrutura facilita automação futura
